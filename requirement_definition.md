1. プロジェクト概要
慶應義塾大学の公式学習管理システム（Canvas/K-LMS）の情報を活用し、学生の日常生活（時間割、課題管理、学内リソースへのアクセス）を効率化するWebアプリケーションを開発する。

2. ターゲット
慶應義塾大学の学部生・大学院生。

3. 技術スタック（推奨）
Frontend: React (Next.js) + Tailwind CSS (モバイルファースト)

Backend: Node.js (またはPython/FastAPI)

Storage: Supabase (Auth, PostgreSQL) ※ユーザーのアクセストークンは暗号化して保存

External API: Canvas LMS API (OAuth連携ではなく、APIトークン入力方式)

4. 実装機能一覧（優先度順）
【フェーズ1：基盤と便利ツール】
ユーザー設定: Canvas APIトークン保存機能（ローカル暗号化）。

課題管理ダッシュボード: 期限順の課題リスト表示、提出済み判定。

時間割機能: 手動入力による時間割作成、今日の授業を表示。

レポテンプレ生成: 所属・学籍番号等の情報をテンプレート化してコピー。

学内情報ハブ: 交通遅延情報の表示。

【フェーズ2：データ活用とLLM連携】
時間割自動生成: ユーザーが登録したコースIDから、Canvas APIを介して授業名・場所を自動同期。

LLMレポート補助: 課題テキストをLLMに投げ、構成案を出力する機能。

教科書マッチング: 教科書の売買掲示板（決済機能なし、マッチングのみ）。

5. データフロー図（プロンプト用補足）
User Action: トークンを入力

App Logic: Canvas APIへリクエスト ➡ JSON取得 ➡ DBへキャッシュ ➡ Frontendで整形表示

6. 非機能要件
セキュリティ: Keio IDおよびパスワードはサーバーで一切保存しない（トークンのみ保存）。

更新頻度: キャッシュを1時間保持し、APIのリクエスト制限（Rate Limit）を回避すること。

プライバシー: ユーザーの成績データや課題内容は、ブラウザのローカルストレージまたは個人のSupabase領域のみで管理する。
